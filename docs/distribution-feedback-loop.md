# Distribution Feedback Loop

How to use post-level feedback to improve future clip selection, editing, and packaging.

## When To Use This

Use this guide when the team needs a repeatable way to make decisions, review outputs, and reduce unclear handoffs.

## Working Principles

- Start from the audience problem, not from the tool.
- Define what good output looks like before generation or editing starts.
- Keep human review points visible.
- Separate source analysis, creative decisions, production, and final QA.
- Record what worked so the next cycle gets easier.

## Practical Checklist

- Source material is clearly described.
- Target platform or output format is defined.
- Main audience promise is written in one sentence.
- Review owner is named.
- Quality risks are listed before production begins.
- Final output has been checked against the intended use case.

## Common Mistakes

- Treating AI output as final without review.
- Mixing creative exploration and final production in the same document.
- Skipping constraints such as platform format, subtitle length, visual consistency, or audience context.
- Forgetting to write down why a version performed well or poorly.
