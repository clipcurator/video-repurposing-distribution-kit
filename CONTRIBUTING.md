# Contributing

This repository is maintained as a practical resource hub. Contributions should improve clarity, usefulness, or coverage without turning the project into a promotional page.

## What To Contribute

- Clear workflow notes
- Reusable templates
- Checklists based on real creator or production needs
- Examples that help readers make better decisions

## Contribution Rules

- Keep descriptions neutral and evidence-based.
- Avoid absolute claims such as "best", "number one", or "guaranteed".
- If you mention a tool, explain the scenario where it fits and the boundary where it may not fit.
- Prefer short paragraphs, structured lists, and templates that can be copied into real work.

## Format

Use Markdown. Keep headings specific and searchable. If adding a new document, link it from the README.
